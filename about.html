About PinnapleUI

The story behind the framework that doesn't suck

The Project

PinnapleUI was born out of frustration with existing C++ GUI frameworks. Too many dependencies, complicated build systems, bloated APIs, and outdated rendering. We wanted something different—something that actually doesn't suck.

Built from the ground up with modern C++ and OpenGL, PinnapleUI focuses on simplicity, performance, and developer experience. No external dependencies (except one tiny header file), no complicated setup, just pure, fast, beautiful UI rendering.

Design Philosophy

  • Zero Bloat - Only what you need, nothing more
  • Immediate Mode - Simple, predictable API that just works
  • Native Performance - Direct Win32 + OpenGL, no middleware
  • Beautiful by Default - Modern dark theme with smooth rendering
  • Developer First - API designed for humans, not machines

Technology Stack

PinnapleUI is built with carefully chosen technologies that prioritize performance and simplicity:

C++17 Win32 API OpenGL stb_truetype Immediate Mode GUI

Why These Choices?

Win32 + OpenGL: Direct access to the OS and GPU means zero overhead. No abstraction layers, no middleware, just pure performance.

stb_truetype: A single-header library for beautiful TTF font rendering. No FreeType, no HarfBuzz, no complexity. Just crisp, antialiased text.

Immediate Mode: Your UI code reads like a story. No state management, no callbacks, no event systems. Just straightforward, predictable code.
